### 7. 当前问题追踪

#### 🔴 问题1：`SetFaceTowardsTargetActor()` 第一次攻击时 Myself 参数无效
- **现象**：怪物第一次被攻击时，函数内部 debug 信息显示 `Myself` 无效；第二次及之后攻击则正常。
- **初步定位**：`TargetActor` 是在怪物的 **Actor 重叠事件**发生时设置的变量，怀疑是重叠事件的触发时机或变量传递有问题。
- **尝试解决**：需要让 GE（Gameplay Effect）携带发起者信息并传递给受击实体。
  - `AddInstigator` 是 C++ 函数，默认未暴露给蓝图 → 改用其他方法。
  - **蓝图替代方案**：使用 **Gameplay Event**，在攻击者蓝图中发送携带信息的事件。

#### 🔴 问题2：使用 Gameplay Event 后，转向仍然错误
- **现象**：每次受到攻击都会转向，但无法正确面对玩家的攻击方向。
- **原因**：尚未定位，可能是事件传递的数据不完整，或者转向函数内部计算逻辑有误。

## 2026-05-22

### 12. GA 读取数据资产与 SetByCaller 流程

#### 当前进展
- GA 已能成功读取数据资产（SkillDataAsset）中的内容 ✅

#### 需求目标
- 让 GE（Gameplay Effect）携带 GA 中的技能数据（如伤害值），通过 `SetByCaller` 传递

#### 完整流程设计
1. 在数据资产中设置伤害值
2. 在 GA 中为伤害值设置对应的 Tag
3. 在 GE 中将 `Health` 修改设置为 `SetByCaller`
4. 在 GA 中制作 GE Spec（Gameplay Effect Spec）并保存到玩家身上
5. 在玩家的 `AttackBox` 检测时，将 GE Spec 塞进 `GameplayEvent` 中（**此步存在潜在问题**）
6. 事件发送给怪物
7. 怪物接收到事件 → Break → 使用 `Instigator` 和 `Context Handle`
8. 怪物受击逻辑：接受事件，判断是否死亡，通过 `Context Handle` 应用 GE，面向攻击者，播放受伤动画

#### 遇到的问题
- **现象**：怪物播放了受伤动画，但没有造成伤害
- **原因**：将 GE Spec 直接“塞进” Event 的方式可能有问题

#### 解决方案（采用新思路）
- 不让 Spec 塞进 Event，而是让**怪物自己去拿**
- 通过 **BPI_Combat 接口**传递 `PendingDamageSpec`
- 新流程：
  1. 怪物收到 `GameplayEvent`
  2. 拿到 `Instigator`（玩家 Actor）
  3. 从玩家 Actor 读取 `PendingDamageSpec`
  4. 调用 `Apply Gameplay Effect Spec to Self`
- **结果**：成功 ✅

## 2026-05-23

### 14. 强制位移与 SetByCaller 向量传递研究

#### 强制位移实现方案探索
- **方案一：作为属性** → ❌ 不可行，强制位移不属于持久化属性
- **方案二：通过 GE 计算类执行** → ❌ 不可行，计算类职责清晰，仅服务于属性和 `SetByCaller` 的 float 值，不允许参与数值计算以外的操作

#### SetByCaller 向量读取问题
- **问题**：蓝图中没有直接读取 `SetByCaller` 值的节点，需要 C++ 实现
- **解决方案**：编写蓝图函数库辅助函数

#### 新增 C++ 辅助函数

| 函数 | 用途 | 状态 |
|------|------|------|
| `static float GetCustomSetByCallerMagnitude(const FGameplayEffectSpecHandle& SpecHandle, FGameplayTag Tag)` | 从 SpecHandle 读取 `SetByCaller` 中设置的 float 值 | ✅ 成功 |
| `static void AssignSetByCallerVector(UPARAM(ref) FGameplayEffectSpecHandle& SpecHandle, FVector Vector)` | 将向量值写入 `SetByCaller`，默认使用 `SetByCaller.Dir.X/Y/Z` 锁定 | ✅ 成功 |
| `static FVector GetCustomSetByCallerVector(const FGameplayEffectSpecHandle& SpecHandle, FGameplayTag XTag, FGameplayTag YTag, FGameplayTag ZTag)` | 从 SpecHandle 读取由三个自定义 Tag 组合的向量 | ✅ 成功 |
| `static FVector GetDefaultSetByCallerVector(const FGameplayEffectSpecHandle& SpecHandle)` | 使用默认 `SetByCaller.Dir.X/Y/Z` 读取向量 | ✅ 成功 |

#### 意义
- 以上函数是读取击退、吸附、击飞等效果数据的基础，现已可用
